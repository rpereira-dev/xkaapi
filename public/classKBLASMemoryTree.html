◆ fetch_list_reduce()

template<int K>
static void KBLASMemoryTree< K >::fetch_list_reduce ( fetch_list_t *  list)
inlinestatic

Given two regions 'a' and 'b' we note 'a -> b' if 'a' and 'b' are consecutive in memory, with 'b' being right-after 'a'. We note a+b the merged (continuous) memory region.

Given a passed list of region L = [a, b, c, d, e] with a->c and e->d, this routine updates the list so that it becomes L = [a+c, b, e+d]

A fetch list element is not only a region, but also has a:

  • a source device
  • a destinatary device which needs to match between elements so they are merged.

Note at this point, the list is a partition of an access, so there cannot be overlap between represented regions
