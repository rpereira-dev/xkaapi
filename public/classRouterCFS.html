Detailed Description

Idea is to use a dijkstra algorithm with weights being BW, but ignoring the link if there is already a pending transfer Problem is it may add significant latency The higher the rank, the lower the performance

const xkrt_router_cfs_map_t RouterCFS::map

A graph where weights[i][j] is the weight of the edge between node 'i' and 'j' If weights[i][j] is UINT8_MAX, then there is no edge If weights[i][j] = weights[i'][j'] / x then the bw on link (i,j) is 'x' times greater than the bw of link (i,j) - i.e. the lower weights[i][j] the greater the BW If used[i][j], then a communication is already occuring on link (i,j)
